# Release Checklist

Use this checklist before publishing a new version of GitHub Repo Polisher.

## Before Release

- Run `npm run lint`.
- Run `npm run build`.
- Test the dashboard with at least two public repositories.
- Check the sample repositories still work offline.
- Update `CHANGELOG.md`.
- Capture new screenshots when the user interface changes.

## Release Notes

Each release note should include:

- What changed.
- Why it matters for maintainers.
- Any known limitations.
- A short verification note.

## Suggested Version Flow

- `0.1.x`: documentation, screenshots, small polish.
- `0.2.x`: scoring presets and export improvements.
- `0.3.x`: example reports and deployment polish.
